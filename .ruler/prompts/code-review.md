# Code Review Prompt

> AI가 코드 리뷰를 수행할 때 사용하는 프롬프트이다.

## 리뷰 수행 방법

변경된 파일을 읽고, 아래 카테고리별로 이슈를 분류하여 리포트한다.
이슈가 없는 카테고리는 생략한다.

---

## 리뷰 카테고리 및 체크리스트

### 1. 타입 안전성 (Critical)

- [ ] `any`, `@ts-ignore`, 이중 캐스팅(`as unknown as`) 사용 여부
- [ ] 타입 가드 없이 `unknown`을 사용하고 있지 않은지
- [ ] 제네릭 타입의 제약 조건이 적절한지
- [ ] API 응답 타입이 실제 응답 구조와 일치하는지

### 2. 아키텍처 (Critical)

- [ ] FSD 레이어 의존성 방향 위반 여부 (`fsd.md` 참조)
- [ ] 파일이 올바른 레이어에 위치하는지
- [ ] barrel export(`index.ts`)를 우회하여 내부 파일을 직접 import하지 않는지
- [ ] 컴포넌트 역할 분리 (ui / container) 준수 여부

### 3. 보안 (Critical)

- [ ] 사용자 입력이 검증 없이 렌더링/사용되지 않는지 (XSS)
- [ ] `dangerouslySetInnerHTML` 사용 시 새니타이즈 여부
- [ ] 민감 정보(토큰, 키)가 코드에 하드코딩되지 않았는지

### 4. 비즈니스 로직 (Major)

- [ ] 비즈니스 로직이 JSX에 섞이지 않았는지 (container로 분리)
- [ ] 엣지 케이스 처리 여부 (빈 배열, null, 0, 빈 문자열)
- [ ] 에러 처리가 적절한지

### 5. 코드 품질 (Minor)

- [ ] 네이밍이 의미 기반인지 (`base.md` 네이밍 규칙 참조)
- [ ] 불필요한 복잡성이 없는지 (과도한 추상화, 미사용 코드)
- [ ] Early Return 패턴 적용 여부
- [ ] 매직 넘버/문자열 대신 상수 사용

### 6. 성능 (Minor)

- [ ] 불필요한 리렌더링을 유발하는 패턴이 없는지
- [ ] 대량 데이터 처리 시 가상화/페이지네이션 고려 여부
- [ ] 이미지 최적화 (lazy loading, 사이즈)

### 7. 테스트 (Minor)

- [ ] 순수 함수에 unit test가 존재하는지
- [ ] `.ui.tsx` 파일에 Storybook이 존재하는지
- [ ] 테스트가 구현이 아닌 동작을 검증하는지

## 리포트 형식

```markdown
## Code Review 결과

### Critical
- [파일명:라인] 이슈 설명 — 수정 제안

### Major
- [파일명:라인] 이슈 설명 — 수정 제안

### Minor
- [파일명:라인] 이슈 설명 — 수정 제안

### Good
- 잘 된 부분에 대한 피드백 (생략 가능)
```

- 이슈는 **구체적 파일명과 라인 번호**를 포함한다.
- Critical은 반드시 수정, Major는 강력 권장, Minor는 개선 제안.
- 칭찬할 부분이 있으면 `Good` 섹션에 포함한다.
